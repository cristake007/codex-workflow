# PROJECTS Workspace Instructions

`PROJECTS` is the shared workspace for both work in progress and delivered projects. Do not assume that a project is temporary, abandoned, or safe to remove based only on its location in this directory.

## Implementation plans

Every implementation plan must be divided into clearly separated tasks or phases.

Each task or phase must:

- have one distinct objective and scope;
- avoid mixing unrelated implementation work;
- state its dependencies, expected changes, validation, and completion criteria;
- leave the repository in a coherent state when completed;
- contain enough explicit context that it can be started in a new Codex chat session without relying on hidden context from an earlier chat.

Do not combine multiple tasks or phases merely to reduce the number of steps. Complete and validate the current task before starting the next one unless the user explicitly requests otherwise.
